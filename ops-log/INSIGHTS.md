# INSIGHTS — プロジェクト改善の累積考察

このファイルは「umashinbun という仕組みをどう良くするか」についての考察を **追記式で積む** 場所。
ニュース内容の感想ではなく、 **fetch の仕組み・ソース構成・選定ロジック・構造化抽出の精度・
事典化・出走予定馬と結果の取得** といったプロジェクト自体の改善に関する所感を蓄積する。

- 日次ログ (`YYYY-MM-DD.md`) の「プロジェクト改善考察」は1日で流れる。 ここはその集約先で **消えない**。
- **書き込むのは週次の改善 routine だけ** ([../IMPROVE_PROMPT.md](../IMPROVE_PROMPT.md))。
  日次 routine は読むのみ。
- 既存の論点は消さず、状態を更新する (`🟢解消` / `🟡継続` / `🔴悪化` / `🔵検証中` のマーカー)。
- 新しい週の考察は **末尾に追記** する (時系列で読めるように)。

## 使い方 (週次 routine 向け)

1. 直近7日の日次ログの「気づき/改善候補」「プロジェクト改善考察」を読む。
2. 下の「論点トラッカー」を更新する (既出論点の状態遷移＋新規論点の追加)。
3. その週に考えたこと・打った手 (PR)・残した宿題を「週次エントリ」に **追記** する。
4. コード変更はあくまで PR で提案 (このファイルへの追記はコミットしてよい)。

### 2026-06-07 — 初期化 (seed)

- このファイルを新設。 umashinbun は kawaraban から派生して10日ほどで設計完了、
  ローカル開発は終わり、 GitHub Actions + Routines の本番運用に入る寸前の状態。
- 設計上は完成しているが、 「実際に routine が動いたら見えてくる課題」 はゼロ。
  論点トラッカーの多くが 🔵検証中 なのは、 routine 起動後の実観測待ちのため。
- **当面の観測ポイント** (初回起動: 2026-06-08 7:00 JST):
  1. race_week モード判定 (宝塚記念 6日前) が正しく走るか
  2. B-0 で netkeiba から planned_horses を取り込めるか
  3. 構造化抽出 (race_id / horses / sires / jockeys / trainers) の非空率がどうか
  4. 過去回顧の馬が `horses` に混入しないか
  5. 新出馬の Wikipedia 事典化が成功するか (シェイクユアハート, タガノデュード等)
- **宿題**: 1週間程度動かして、 ops-log を見ながら「設計通り動かないところ」を特定し、
  次回 (2026-06-15 月曜) の週次改善 routine で改善PRを出す。

### 2026-06-14 — 初週運用完了・宝塚記念週総括

**総評**: 設計の妥当性は概ね確認できた1週間。race_week モードが 8 日連続で安定稼働し、構造化抽出・事典化・結果取得の主要フローが全て初実績を得た。一方で「ログから繰り返し浮かんだ穴」が 2 本のPRに収束した。

**解消した論点（全 5 件を 🔵検証中 → 🟢解消）**:
1. fetch-runners.ts 不使用（WebFetch代替が安定）
2. planned_horses 毎朝最新化（B-0が 8 日間機能・6/11 に 17→18 頭更新）
3. 結果取得 B-0-2（宝塚記念で初の実運用成功）
4. horses-profile.json 事典化（宝塚記念 18 頭全員登録済み）
5. race_id/horses への過去回顧馬混入（8 日間ゼロ件）

**新たに見えた穴**:
- **netkeiba 本文取得が全 8 日ゼロ成功**: JS動的レンダリングで構造的に取得不能。description（300〜600字）で暗黙的に代替しているが ROUTINES_PROMPT に未記載。→ PR で明示化。
- **news カテゴリがソースゼロ**: 業界ニュースが偶発的にしか拾えない。この環境から URL 検証ができないため次回以降の課題。
- **overseas 86 件/日が race_week で毎日全除外**: 宝塚記念週中に英ダービー・ベルモントS・ロイヤルアスコットが開催されたが取りこぼし。→ PR で条件付き採用ルールを追加。

**打った手（PR `improve/feeds-2026-06-14`）**:
1. ROUTINES_PROMPT.md: overseas の条件付き採用ルール（固定G1リスト AND 日本馬名、1日上限2件）
2. ROUTINES_PROMPT.md: netkeiba 本文取得失敗時のフォールバック明示

**残した宿題**:
- news カテゴリへのソース追加（URL 検証が先、GH Actions 環境で確認）
- 調教タイム専用ソース（公開 RSS なし・スクレイプは工数大）
- 競馬ラボが 1 件/8 日と少ない問題（Yahoo!配信タイミング依存の可能性、次回 race_week 以外で計測）
- ファミリータイム・ミクニインスパイアの Wikipedia 再試行（宝塚記念後に major_results が更新されれば自動付与、記事から再試行）
